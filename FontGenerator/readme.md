# FontGenerator

A utility for converting bitmap images into FIGlet font files (.flf).

## Overview

FontGenerator reads a 128x128 pixel bitmap image containing a 16x16 grid of 8x8 pixel characters and generates FIGlet font files. It produces two output formats:

- **Standard** - Each output character represents 2 vertical pixels using Unicode half-block characters
- **Condensed** - Each output character represents a 2x2 pixel grid using Unicode quadrant block characters

## Requirements

- .NET 8.0 SDK (Windows)
- `System.Drawing.Common` package

## Building

```bash
dotnet build FontGenerator/FontGenerator.csproj
dotnet run --project FontGenerator
```

## Usage

1. Create a 128x128 pixel bitmap image with characters arranged in a 16x16 grid
2. Each cell is 8x8 pixels representing one character (ASCII 0-255)
3. Place the bitmap in the same directory as the executable
4. Run the program

The generator currently uses hardcoded filenames:
- Input: `charmap-inverted.png`
- Output: `bmp-inverted.flf` (standard) and `bmp-inverted-condensed.flf` (condensed)

## Bitmap Format

```
+---+---+---+---+- ... -+---+
| 0 | 1 | 2 | 3 |       | 15|  Row 0 (chars 0-15)
+---+---+---+---+- ... -+---+
| 16| 17| 18| 19|       | 31|  Row 1 (chars 16-31)
+---+---+---+---+- ... -+---+
 ...
+---+---+---+---+- ... -+---+
|240|241|242|243|       |255|  Row 15 (chars 240-255)
+---+---+---+---+- ... -+---+
```

- Total size: 128x128 pixels
- Character grid: 16x16 characters
- Each character cell: 8x8 pixels
- Black pixels (brightness < 128) = filled
- White pixels (brightness >= 128) = empty

## Output Formats

### Standard Format (2-pixel representation)

Converts each column of 2 vertical pixels into a Unicode block character:

| Top | Bottom | Character      |
|-----|--------|----------------|
| ■   | ■      | █ (full block) |
| ■   | □      | ▀ (upper half) |
| □   | ■      | ▄ (lower half) |
| □   | □      | (space)        |

Result: 8 characters wide × 4 lines tall per glyph

### Condensed Format (4-pixel representation)

Converts each 2x2 pixel grid into a Unicode quadrant block character:

| Pattern | Character | Description             |
|---------|-----------|-------------------------|
| □□/□□   | (space)   | Empty                   |
| □□/□■   | ▗         | Bottom-right            |
| □□/■□   | ▖         | Bottom-left             |
| □□/■■   | ▄         | Bottom half             |
| □■/□□   | ▝         | Top-right               |
| □■/□■   | ▐         | Right half              |
| □■/■□   | ▞         | Diagonal                |
| □■/■■   | ▟         | All except top-left     |
| ■□/□□   | ▘         | Top-left                |
| ■□/□■   | ▚         | Reverse diagonal        |
| ■□/■□   | ▌         | Left half               |
| ■□/■■   | ▙         | All except top-right    |
| ■■/□□   | ▀         | Top half                |
| ■■/□■   | ▜         | All except bottom-left  |
| ■■/■□   | ▛         | All except bottom-right |
| ■■/■■   | █         | Full block              |

Result: 4 characters wide × 4 lines tall per glyph

## Generated FIGlet Header

```
flf2a$ <height> <height> <max_width> -1 4
Generated by Bitmap to FIGfont Converter
<description>
Characters are 8x8 pixels from a 128x128 bitmap
Characters are arranged in a 16x16 grid in the bitmap
```

## Example Output

Standard format renders the letter 'A' as:

```
▄▀▀▀▄@
█▀▀▀█@
█   █@
▀   ▀@@
```

Condensed format renders the same 'A' as:

```
▗▀▀▖@
█▀▀█@
█  █@
▀  ▀@@
```
